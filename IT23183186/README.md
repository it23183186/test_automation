## Usage

### Running the Automated Tests
```bash
# Navigate to the project directory
cd test_automation

# Run the automation with default settings
python test_automation.py --excel "Assignment 1 - Test cases.xlsx" \
  --url "https://www.pixelssuite.com/chat-translator" \
  --wait-ms 5000 \
  --type-delay-ms 80 \
  --slow-mo-ms 200 \
  --save-every 1 \
  --keep-open
```

### Command Parameters:
- `--excel`: Path to the Excel file containing test cases (default: auto-detected)
- `--url`: URL of the application to test (default: https://www.pixelssuite.com/chat-translator)
- `--wait-ms`: Wait time between operations in milliseconds (default: 5000)
- `--type-delay-ms`: Delay between typing characters in milliseconds (default: 30)
- `--slow-mo-ms`: Slow-motion effect for debugging in milliseconds (default: 0)
- `--save-every`: Save Excel file after every N test cases (default: 1)
- `--keep-open`: Keep browser open after tests complete

### Output
After execution, the Excel file will be automatically updated with:
- **Actual output:** The Sinhala output generated by the system
- **Status:** PASS, FAIL, or ERROR
- **Singlish input types covered:** Categorization of the input type(s)
- **Evidence/Rationale:** Explanation of why each input type was selected

## Test Results Summary
- **PASS Cases:** 4 (cases where system correctly converted input)
- **FAIL Cases:** 43 (cases where system failed to convert correctly)
- **ERROR Cases:** 0 (cases with UI interaction errors)

### Key Findings:
The system demonstrates partial support for English word insertions and place names but has significant gaps in:
- Complex punctuation handling
- Spelling variant consistency
- Multi-word English phrase handling
- Complex emoji combinations
- Very long inputs (character limit issues)

7. **Singlish input types covered** - Categories of input types
